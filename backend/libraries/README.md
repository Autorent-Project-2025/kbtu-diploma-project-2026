# Backend Libraries

## Назначение
`backend/libraries` содержит переиспользуемые backend-библиотеки.

Это не сервисы:
- у библиотек нет собственного контейнера в `docker-compose.yml`;
- они не владеют БД и не запускаются отдельно;
- их задача - давать общий код, который подключают несколько сервисов.

## Что сейчас лежит в каталоге
| Каталог | Назначение |
|---|---|
| `messaging-dotnet` | Общая .NET-библиотека для RabbitMQ topology, envelope, publisher и контрактов интеграционных событий |

## Когда добавлять код в `libraries`
Кладите код сюда, если одновременно выполняются условия:
- он нужен нескольким backend-сервисам;
- это именно библиотека, а не самостоятельный runtime-компонент;
- у кода есть стабильная граница ответственности;
- перенос уменьшает дублирование строковых констант, контрактов и инфраструктурного boilerplate.

## Когда не надо использовать `libraries`
Не кладите сюда:
- бизнес-логику одного конкретного сервиса;
- контроллеры, hosted services, миграции и persistence конкретного bounded context;
- код, который нужен только Node/TypeScript-сервисам;
- общий каталог "на всякий случай" без явного владельца и сценария повторного использования.

## Правило зависимости
Библиотеки из `backend/libraries` могут зависеть от пакетов и абстракций, но не должны зависеть от конкретных сервисов из `external`, `internal` или `shared`.
